ハンディ（BT-A500）モック v0.10

タブレット v0.10 の仕様（保留時スキャン状態保持）をベースに、KEYENCE BT-A500（Android／3.5inch 480×800／物理キー＋レーザースキャナ）向けに再構成したモックです。v0.3 では実機の物理キー配列（F1-F4／5方向D-pad／テンキー／側面トリガー／メイン・トリガー／ESC・TAB・音量・電源）を全て配置し、画面上のボタンとタッチ操作を最小化しています。

🎮 デモ操作

🔑 物理キー割当（v0.3 拡張）

メイン トリガー（前面・大）／側面トリガー L／R：スキャン
　待機画面：ピッキング№バーコード読取（モックではダミー伝票読込） ／ 検品中：商品スキャン ／ 最終チェック画面：納品書バーコードで自動完了（サンドイッチ方式）
F1待機＝連絡事項を再表示 F2検品＝強制OK ／ 待機＝QRラベル再発行 F3同梱物を1件ずつON（次の未チェック） F4📋 一括検品（残数確認後・全件完了扱い→最終チェック画面へ） ▲▼検品行カーソル移動（青枠＝選択中） ENT選択行を +1スキャン／モーダルでは「確定」 ◀▶同梱トグル（▶＝次へON／◀＝直近をOFF） 0〜9数量入力（直接打鍵で数量モーダル自動起動） CLR数量入力をクリア ←(BS)数量入力を一文字削除 TAB次の検品行へカーソル移動 ESCモーダル閉じる／中断（待機へ戻る） VOL+/-操作音量調整（モック） ⏻電源（モック）

🔧 タブレットとの差異（ハンディ仕様）

・特殊梱包QR印刷は実行しない（対象伝票はタブレット梱包ステーションで再スキャン要）
・画面幅が狭いため、ピッキング情報は上部バー／商品リスト／下部同梱の3段構成
・同梱物は画面下にコンパクト表示（F3で順次チェック）
・数量キーパッドは画面全体をほぼ使う大型配置（誤タップ回避）
・左上「PK-NO」バーにピッキングNoを常時表示（現場での口頭確認対応）

📝 動作確認手順（v0.10 サンドイッチ方式）

1. 起動 → 連絡事項4件を☑ → 「作業開始」（ENTでも可）
2. 「① 一般顧客」を読込（＝ピッキング№バーコードスキャン）→ メイントリガーまたはL/R側面トリガーで商品スキャン
3. ▲▼ で検品行カーソル移動（青枠表示）→ ENTで選択行を +1スキャン
4. 数量が多い商品：0〜9 を直接押下で数量モーダル自動起動／CLR/←(BS)/ENTで操作
5. F3 や ▶ で同梱物を順次チェック
6. 全点完了 → 自動的に最終チェック画面が開く（F4押下不要）→ ☑ → 納品書バーコードをスキャン → ✓ 梱包完了 トースト表示 → 完了画面
6'. 検品を省略したい場合 → F4（⏭検品省略）→ 残数確認 → 自動で最終チェックへ → 納品書スキャンで完了
0. 「✓ 検品済み 再スキャン」でピッキング№再読込ポップアップ（確認 / 検品戻し）
7. 待機戻り → 「③ 冷凍・特殊梱包」を読込 → 完了時に「タブレット梱包ステーションへ回送」表示
8. 待機画面 → F2（または「QR再発行」）→ (デモ)スキャン → 発行

※ 物理キー操作は PC キーボードでも検証可能（実機ではすべて物理キー、画面タッチは最小化）

　- 同じ納品書を再スキャンすると「保留中」ポップアップ表示
　- 「▶ 続きから」で前回状態を完全復元 ／ 「↻ 最初から」で破棄
・本部連絡（📢）も同じ仕組みで保留（指示待ち中も検品済み数量保持）

v0.7 ★一括検品＋保留メニュー追加

・中断（ESC/btnBack）に保留メニュー追加：従来の即破棄から、「⏸ 検品保留」「📢 本部連絡」「🚫 中断（破棄）」 の3択に拡張
　- 本部連絡：4分類ボタン（🎁のし / 📦商品 / ✏入力内容 / 🌐WEB）+ 補足テキスト
　- 送信先：管理PC「連絡」タブ → 着信バッジで通知
　- ペイロード：from / pkNo / invoice / category / body / sentAt / read（未読フラグ）

v0.6 ★自動最終チェック＋検品省略
・v0.5 で要件解釈ミスがあったため再仕様化
・商品検品完了で「最終チェック画面が自動展開」（F4押下を省略）
　- 全商品スキャン完了で 0.35秒後に自動でモーダルが開く
・F4を「⏭ 検品省略」用途に転用（オレンジ色／検品省略表示）
　- 押下→残数確認ダイアログ→OK→全件完了扱い→自動で最終チェック画面へ
　- 強制OK（F2）と異なり理由コード不要（検品の流れそのものを省略する用途）
　- 監査ログには「検品省略・梱包完了 残N点を省略」を記録
・F2=強制OK は別系統として維持（理由コード必須）

・最終チェック画面に「✓ 商品検品状況」セクション追加：全アイテムの検品状況と強制OK状況を一画面で確認
・のし関連の同梱物項目を表示から除外（上部のし表示と重複しないよう "熨斗"/"のし"/"台紙" を含む koudoku/instructions は描画スキップ）
・確定ボタンを「✓ 梱包完了」に変更
・フロー: 「①納品書 → ②商品 → ③📋最終Chk」に簡素化

v0.4 ★納品書トリガー＋同梱物モーダル＋検品戻し
・トリガーを「納品書バーコード」に一本化：v0.3までのサンドイッチ検品（①ピッキングNo→②商品→③納品書）を撤廃。納品書1枚で①読み込み →②商品検品 →③同梱物確認 →完了。
・同梱物モーダル：商品検品完了後（F4＝梱包完了）に表示。のし種類・名称（基幹データ）と同梱指示（補助マスタ／取込なし）を分けて☑可能。全☑で完了画面へ進行。
・検品済み再スキャン ポップアップ：完了済みの納品書を読むと「✓ 検品済み」と表示。「確認」で待機戻り、「検品戻し」で完了状態を解除→再検品（操作ログ記録）。

v0.3 ★物理キー対応
・物理キー配列を BT-A500 実機準拠 に拡張（F1-F4／5方向D-pad／12キー数字パッド／L・R 側面トリガー／メイン・トリガー／ESC・TAB・VOL±・電源）
・D-pad ▲▼ で検品行カーソル移動（青枠）→ ENT で選択行を +1 スキャン。タッチ不要で全行スキャン可能に
・テンキー 0〜9 直接打鍵で数量モーダルを自動起動（カーソル行 or 次未完行が対象）。CLR/←(BS)/ENT で確定までキーボードのみで完結
・◀▶で同梱物を双方向トグル（F3は従来どおり「次の未チェック」）
・側面トリガー L／Rを追加（実機ではどちらでもスキャン可能を再現）
・全キーに押下時の黄色フラッシュ視覚フィードバックを追加（120ms）
・モーダル内では ENT＝確定／ESC＝キャンセル／0-9＝入力／BS＝削除／CLR＝クリア に統一
・物理キーはすべて onclick + キーボード両方で動作（hwKey ハンドラに統一）
