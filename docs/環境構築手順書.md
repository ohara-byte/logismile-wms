## 3. Docker Compose（PostgreSQL）のセットアップ

### 3-1. docker-compose.yml 作成

プロジェクトルート（`wms/`）に以下のファイルを作成:

```yaml
# docker-compose.yml
version: '3.8'

services:
  db:
    image: postgres:16
    container_name: wms_db
    environment:
      POSTGRES_USER: wms_user
      POSTGRES_PASSWORD: wms_password_change_me
      POSTGRES_DB: wms_db
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    restart: unless-stopped

  pgadmin:
    image: dpage/pgadmin4
    container_name: wms_pgadmin
    environment:
      PGADMIN_DEFAULT_EMAIL: admin@wms.local
      PGADMIN_DEFAULT_PASSWORD: admin123
    ports:
      - "5050:80"
    depends_on:
      - db
    restart: unless-stopped

volumes:
  postgres_data:
```

### 3-2. データベース起動

```bash
# Docker コンテナ起動
docker compose up -d

# 起動確認
docker compose ps
# NAME          STATUS    PORTS
# wms_db        running   0.0.0.0:5432->5432/tcp
# wms_pgadmin   running   0.0.0.0:5050->80/tcp
```

> 💡 **pgAdmin** は DB の中身をブラウザで確認できるツールです。  
> `http://localhost:5050` にアクセス → admin@wms.local / admin123 でログイン

## 8. 本番デプロイ（オンプレミス）

### 8-1. 本番用 Docker Compose

```yaml
# docker-compose.prod.yml
version: '3.8'

services:
  app:
    build: .
    container_name: wms_app
    environment:
      DATABASE_URL: postgresql://wms_user:${DB_PASSWORD}@db:5432/wms_db
      NEXTAUTH_SECRET: ${AUTH_SECRET}
      NEXTAUTH_URL: http://${SERVER_IP}
      NODE_ENV: production
    ports:
      - "80:3000"
    depends_on:
      - db
    restart: always

  db:
    image: postgres:16
    container_name: wms_db_prod
    environment:
      POSTGRES_USER: wms_user
      POSTGRES_PASSWORD: ${DB_PASSWORD}
      POSTGRES_DB: wms_db
    volumes:
      - /data/wms/postgres:/var/lib/postgresql/data
    restart: always
```

### 8-2. Dockerfile

```dockerfile
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npx prisma generate
RUN npm run build

FROM node:20-alpine AS runner
WORKDIR /app
COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static
COPY --from=builder /app/public ./public
EXPOSE 3000
CMD ["node", "server.js"]
```

### 8-3. 本番起動

```bash
# .env.production を作成（本番用の強いパスワードに変更）
cp .env.local .env.production
# ← パスワード・シークレットを変更してください

# ビルド & 起動
docker compose -f docker-compose.prod.yml up -d

# マイグレーション（初回のみ）
docker exec wms_app npx prisma migrate deploy
```

## 9. よくあるトラブルと解決策

| 症状 | 原因 | 解決策 |
|------|------|--------|
| `prisma: command not found` | パスが通っていない | `npx prisma` を使う |
| DB接続エラー | Dockerが起動していない | `docker compose up -d` |
| ポート5432使用中 | 他のPostgreSQLが起動中 | 既存サービスを停止 |
| `NEXTAUTH_SECRET` エラー | 環境変数が設定されていない | `.env.local` を確認 |
| スキャンが反応しない | Web Serial API非対応 | Chrome / Edge を使用 |

## 10. ★ プリンター（SCeaTa CT4-LX）の設定

### 10-1. プリンターのIPアドレス設定

各プリンター（6台）に固定IPアドレスを割り当てます。

```
推奨IPレンジ例:
PRT-01: 192.168.10.101  倉庫1F検品ステーション
PRT-02: 192.168.10.102  倉庫1F検品ステーション2
PRT-03: 192.168.10.103  倉庫2Fスイーツ
PRT-04: 192.168.10.104  倉庫2F冷凍
PRT-05: 192.168.10.105  ハンディゾーン
PRT-06: 192.168.10.106  予備（朝礼スペース）
```

各プリンターは SCeaTa の管理ツール / 本体ボタンから IP・サブネットマスク・ポート（標準9100）を設定します。

### 10-2. プリンターマスタへの登録

管理PCから「マスタ管理 → プリンターマスタ」で全6台を登録します。

```
コード / 名称 / IPアドレス / ポート / モデル / 配置場所 / ラベルサイズ
PRT-01 / 倉庫1F-01 / 192.168.10.101 / 9100 / SCeaTa CT4-LX / 倉庫1F検品ステーション / 30x40
...
```

### 10-3. ★ 端末×既定プリンターの紐付け

「マスタ管理 → 端末-プリンター紐付け」で各端末（タブレット10台 / ハンディ5台）の既定出力先を設定します。

```
端末コード / 既定プリンター
TAB-01 / PRT-01
TAB-02 / PRT-01
HND-01 / PRT-05
HND-02 / PRT-05
...
```

> **★ 端末側にはプリンター設定画面はありません。** ログイン時にサーバから既定プリンターを取得して使用します。

### 10-4. 印刷テスト

```bash
# 接続テスト（サーバから各プリンターへping）
ping 192.168.10.101

# 試し印刷API
curl -X POST http://localhost:3000/api/print/qr \
  -H "Content-Type: application/json" \
  -d '{"pkNo":"PK-TEST-0001","deviceCode":"TAB-01"}'
```

### 10-5. ラベル用紙

- **SATO レスプリ・シータラベル 超高感度サーマルB 30×40mm**
- 印字内容：QRコード（納品書№） + その下に納品書№テキスト

## 11. ★ 端末（タブレット・ハンディ）のセットアップ

### 11-1. タブレット（HP14・10台）

```
1. Wi-Fi 設定（社内ネットワーク）
2. Chrome をインストール（プリインストール済みの場合は最新版に）
3. 開発用URL（または本番URL）をブックマーク
4. ブックマークをホーム画面に追加（PWA風アクセス）
5. 自動スリープを長めに設定（作業中に画面が消えないよう）
6. タブレットマスタ（devices テーブル）に端末コード（TAB-01〜TAB-10）を登録
7. 端末-プリンター紐付けで既定プリンター設定
```

### 11-2. ハンディ（KEYENCE BT-A500・5台）

```
1. Wi-Fi 設定（社内ネットワーク）
2. Chrome をインストール
3. スキャナ動作確認（HID キーボードエミュレーション）
4. 設定でEnter送信を有効に（Enterキーで連絡事項☑切替に必要）
5. 開発用URL（または本番URL）をブックマーク
6. ハンディマスタ（devices テーブル）に端末コード（HND-01〜HND-05）を登録
7. 端末-プリンター紐付けで既定プリンター設定
```

### 11-3. 動作確認

```
1. タブレット/ハンディから https://wms-server/handy または /tablet にアクセス
2. 社員番号（emp_code）を入力してログイン
3. ピッキング№スキャン → 商品スキャン → 納品書№スキャン → 完了
4. QR印刷フラグONの伝票でラベル印刷を確認
5. 管理PCのダッシュボードでリアルタイム反映を確認
```
