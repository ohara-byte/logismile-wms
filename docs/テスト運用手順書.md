## 1. 初回セットアップ

### 1-1. 強い認証シークレットの確認

`.env.production.local` の以下が **本番用ランダム値** になっていることを確認：

- `NEXTAUTH_SECRET` — 64 文字の hex
- `DATABASE_URL` — Postgres の本番パスワード
- `INTRANET_CIDR_LIST` — 社内 LAN の CIDR

このファイルは Git にコミットされません（.gitignore 対象）。
変更したら必ず `npm run build` を再実行。

### 1-2. ファイアウォール設定（管理者権限が必要）

PowerShell を **管理者として** 開いて：

```powershell
# 社内 LAN（192.168.1.0/24）からのみ ポート 3000 を許可
New-NetFirewallRule -DisplayName "WMS HTTP" `
  -Direction Inbound -Protocol TCP -LocalPort 3000 `
  -RemoteAddress 192.168.1.0/24 `
  -Action Allow

# pgAdmin（管理用、社内 LAN のみ）
New-NetFirewallRule -DisplayName "WMS pgAdmin" `
  -Direction Inbound -Protocol TCP -LocalPort 5050 `
  -RemoteAddress 192.168.1.0/24 `
  -Action Allow
```

### 1-3. PC のスリープを無効化（必須）

PC がスリープすると全端末がログイン不能になるため：

1. 設定 → システム → 電源 & バッテリー → 画面とスリープ
2. 「電源接続時、次の時間が経過した後にデバイスをスリープ状態にする」を **「なし」**

### 1-4. 初期 admin ユーザーのパスワード変更（推奨）

seed の初期パスワード `admin123` を本番運用前に強いものへ変更：

```powershell
# パスワードを bcrypt 化（例: 新しいパスワード = SecureP@ss2026）
node -e "console.log(require('bcryptjs').hashSync('SecureP@ss2026', 10))"

# pgAdmin か psql で users テーブルを更新
docker exec -it wms_db psql -U wms_user -d wms_db -c `
  "UPDATE users SET password_hash='<上で生成したhash>' WHERE email='admin@wms.local';"
```

## 2. 日々の起動・停止

### 2-1. 手動起動

`scripts\start-prod.bat` をダブルクリック。
コマンドプロンプトが開いたまま動き続けます（このウィンドウを閉じると停止）。

### 2-2. 手動停止

`scripts\stop-prod.bat` をダブルクリック。

### 2-3. 自動起動（推奨：タスクスケジューラ）

「タスクスケジューラ」を開いて：

1. **タスクの作成**
2. 全般タブ：
   - 名前: `WMS-AutoStart`
   - 「ユーザーがログオンしているかどうかにかかわらず実行」
   - 「最上位の特権で実行する」にチェック
3. トリガータブ：
   - 新規 → 「スタートアップ時」
4. 操作タブ：
   - 新規 → プログラム/スクリプト：`C:\Users\revex\LojiSmaile\wms\scripts\start-prod.bat`
   - 開始（オプション）：`C:\Users\revex\LojiSmaile\wms`
5. 条件タブ：
   - 「コンピューターを AC 電源で使用している場合のみタスクを開始する」のチェックを **外す**

これで PC 起動時に WMS も自動起動します。

## 3. 日次バックアップ（推奨設定）

### 3-1. 手動バックアップ

`scripts\backup-db.bat` をダブルクリック。
`data\backups\wms_db_YYYYMMDD.sql` が生成されます。

### 3-2. 自動バックアップ

タスクスケジューラで以下のタスクを追加：

| 設定 | 値 |
|------|-----|
| 名前 | `WMS-DailyBackup` |
| トリガー | 毎日 0:00 |
| 操作 | `C:\Users\revex\LojiSmaile\wms\scripts\backup-db.bat` |
| 開始 | `C:\Users\revex\LojiSmaile\wms` |

7 日以上前のバックアップは自動削除（スクリプト内で）。
保存期間を変えるには `forfiles -d -7` の `7` を変更してください。

### 3-3. バックアップから復元

```powershell
# 起動中の Postgres にリストア
docker exec -i wms_db psql -U wms_user -d wms_db < data\backups\wms_db_YYYYMMDD.sql
```

⚠️ 既存データが上書きされます。事前に DB を空に（`prisma migrate reset`）するか、別 DB に復元してから移行してください。

## 7. トラブルシュート

### Q. ブラウザで表示されない

1. このPC で `http://localhost:3000` は開けるか？
   - 開けない → `start-prod.bat` を再実行
2. 別 PC で `http://192.168.1.139:3000` は開けるか？
   - 開けない → ファイアウォール設定を確認（手順 1-2）
   - LAN IP が変わっていないか `ipconfig` で確認

### Q. ログインできない

- パスワード違い → 上記初期ログイン情報を試す
- 「試行回数を超過しました」(429) → 15 分待つ or `node -e "console.log('reset rate limit by restart')"` で再起動

### Q. 「社内ネットワークからのみアクセス可能」(403)

- アクセス端末の IP が `INTRANET_CIDR_LIST` に含まれていない
- `.env.production.local` の `INTRANET_CIDR_LIST` を確認、必要に応じて追加して `npm run build`

### Q. プリンターから印刷されない

- 現状 `PRINTER_DRY_RUN=true` のため、印刷ジョブは送信されません（ログのみ）
- 実機接続後、`.env.production.local` で `PRINTER_DRY_RUN=false` に変更
- printers テーブルの `ip_address` が実機と一致しているか確認
- 端末 → device_printer_map → printer の紐付けが正しいか確認

### Q. ポート 3000 が他のプロセスで使われている

```powershell
netstat -ano | findstr :3000
taskkill /PID <該当 PID> /F
```

## 8. 本番移行前のチェックリスト

テスト運用 1〜2 ヶ月後、本番移行前に：

- [ ] DB バックアップが日次で取れているか
- [ ] 強制OK / 未マップ商品 / JAN 不備のアラートが妥当な頻度か
- [ ] 平均梱包時間が現運用と乖離していないか（レポート → サマリー）
- [ ] 終了予定時刻の精度が実態と合うか
- [ ] 1 ヶ月分の検品データで更新された std_min / skill_coefficient が現実的か
- [ ] 全担当者がモバイル端末でログインできるか
- [ ] 全プリンターから QR ラベル印刷ができるか（PRINTER_DRY_RUN=false 時）
- [ ] 削除/復活機能の運用フロー確認
- [ ] 本番サーバーへの移設手順（DB ダンプ → 別マシン restore）の検証
