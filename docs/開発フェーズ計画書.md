```

Phase 2 │    │▓▓▓▓│                         基幹連携（Thomas取込）+ IFアダプタ層 + JAN検証
```

## Claude Code との作業フロー

### 基本的な進め方

```
1. Claude Code を起動（ターミナルで `claude` コマンド）
2. プロジェクトルートに移動（CLAUDE.md があるディレクトリ）
3. フェーズ・タスク番号を指定して指示する

例:
「Phase 2 のタスク 2-3 を実装してください。
 src/lib/jan-validator.ts を作成し、
 - 空欄チェック
 - 13桁チェック
 - チェックデジット検証
 を実装してください。重複チェックは含めないでください（重複は許容仕様）。
 ユニットテストも作成してください。」
```

### 効果的な指示の書き方

```
✓ 良い指示:
「CLAUDE.md / 要件定義書_WMS_v1.5.docx の 7章を参照して、
 src/lib/integration/csv-adapter.ts を実装してください。
 表7.4-1（商品マスタ）と 表7.4-2（出荷指示）のマッピングを反映し、
 JAN検証は重複を許容、ピッキング№重複はエラーとしてください。」

✗ 避けるべき指示:
「CSVを取り込む処理を作って」（曖昧すぎる）
```

### ファイルの渡し方
- `CLAUDE.md` は Claude Code が自動で読みます
- `要件定義書_WMS_v1.5.docx` / `DB設計書.md` / `API仕様書.md` も同じディレクトリに置いておく
- モックHTMLは `添付して参照させる` か `ファイルパスを指定`
- GPシフトCSV・出荷指示CSVは `data/sample_csv/` に配置して参照
